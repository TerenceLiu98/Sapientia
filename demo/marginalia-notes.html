Journal of Interface Thought / 2026

Reading systems should let commentary accumulate beside the sentence, not above it.

Scholarly notes become more alive when they remain visibly attached to the line that provoked them. A conventional sidebar often treats a note as a database object, while marginalia behaves like a situated response: an afterthought, a challenge, a synthesis, or a whispered cross-reference.

The geometry matters. When the reader can still sense page position, the note inherits the memory of the source passage. This makes return visits faster, because the thought is recalled together with the visual neighborhood in which it was written.

A successful margin does not overwhelm the document. Instead it offers a light structure for response: one living note in focus, several quieter slips nearby, and just enough metadata to re-enter context without turning the act of reading into project management.

Anchored, not grouped

Page markers behave like quiet folio marks on a rail. The sidebar reads as a continuation of the page edge, not as a separate list manager.

Excerpt, not metadata

Folded notes show the thought itself. Source tags and status remain visible, but they support the sentence rather than replacing it.

One live note

Only one note expands fully on the active page. This keeps the margin legible and preserves the feeling of reading beside the text instead of managing many open documents.
